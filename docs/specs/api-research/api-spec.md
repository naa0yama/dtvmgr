### db.php 放送時間データの取得(ProgLookup)

Ref: https://docs.cal.syoboi.jp/spec/db.php/

> [!TIP]
> 番組表ではサブタイトルが表示されているのに、db.phpの結果のSubTitleが空になっていることがあります。
> 番組表と同じサブタイトルを表示するためには、別途サブタイトルテーブルからTIDとCountで検索するか、 JOIN=SubTitleパラメータを追加してSTSubTitleを利用します。
> サブタイトルテーブルはタイトルデータのSubTitlesをパースして生成されています。
> このため、サブタイトルが修正されても放送時間データのLastUpdateは更新されません。

| フィールド    | 型        | 説明                             | 例                    |
| ------------- | --------- | -------------------------------- | --------------------- |
| `PID`         | `u32`     | 番組ID                           | `687485`              |
| `TID`         | `u32`     | タイトルID                       | `7667`                |
| `StTime`      | `String`  | 放送開始時刻                     | `2026-02-03 23:00:00` |
| `StOffset`    | `i32?`    | 開始オフセット (秒)              | `0`                   |
| `EdTime`      | `String`  | 放送終了時刻                     | `2026-02-03 23:30:00` |
| `Count`       | `u32?`    | 話数 (0=特番/未設定)             | `5`                   |
| `SubTitle`    | `String?` | サブタイトル (空の場合あり)      | _(empty)_             |
| `ProgComment` | `String?` | 番組コメント                     | _(empty)_             |
| `Flag`        | `u32?`    | フラグ (bitmask, 2=初回等)       | `0`                   |
| `Deleted`     | `u32?`    | 削除フラグ                       | `0`                   |
| `Warn`        | `u32?`    | 警告フラグ                       | `0`                   |

### db.php 放送時間データの取得(ProgLookup) Bulk

一括で取得する場合、batch 処理などでは1件ずつファイル名から時間を指定してリクエストしていたらリクエスト数が莫大になるので Bulk としてリクエストして処理する方が好ましい

> [!TIP]
> 2009-04-22現在112,000件のデータがありますが、1回のリクエストで最大で5,000件しか出力しません。
> すべてのデータを取り出すにはStTime、Range、LastUpdateを使って対象を絞り込みリクエストを複数回に分ける必要があります。
> ちなみに2008年は全体で28,000件、4月は2,335件でした。

# dtvmgr-vmaf: search モジュール

> 親ドキュメント: [Architecture.md](./Architecture.md)

## 概要

補間二分探索アルゴリズムの実行ループを実装する。目標 VMAF を達成する最大の `q` 値(= 最小ファイルサイズ)を探索する。

## アルゴリズム

### 初期値

探索の開始点は `EncoderConfig.quality_hint` から算出する。中間点ではなくベンチマーク実測値を起点とすることで、典型的なケースでは 2-3 イテレーションで収束する。

### 探索ループ

各イテレーションで以下を実行する:

1. 候補 `q` 値で全サンプルをエンコードし、平均 VMAF を計測
2. VMAF が目標以上の場合:
   - 隣接 `q+1` が未テストなら次のイテレーションでテスト(ファイル縮小方向に貪欲探索)
   - 隣接 `q+1` がテスト済みで目標未達なら、現在の `q` が最適 → 収束
3. VMAF が目標未満の場合:
   - 目標以上の観測点があれば `vmaf_lerp_q` で補間して次の `q` を決定
   - なければ `min_q` 方向にステップ(区間二等分)

### 収束条件

- 目標 VMAF を達成する `q` の隣接 `q+1` が目標未達であることを確認した時点で収束
- `max_q` 境界で目標達成した場合も収束
- サイズ制約(`max_encoded_percent`)も同時にチェックする

### 重複回避

テスト済みの `q` 値は `attempts` ベクタで管理し、同じ `q` を再テストしない。重複候補が選ばれた場合は `q+1` または `q-1` の未テスト値に調整する。

## MAX_ITERATIONS

最大 `12` イテレーションで探索を打ち切る。ヒント値起点の補間探索により、通常は 3-5 イテレーションで収束するため、この上限に達するのは異常ケースのみ。

## select_best_result(フォールバック)

探索ループが収束せずに終了した場合、以下の優先度で結果を選択する:

1. **目標達成**: VMAF >= `target_vmaf` かつサイズ制約内の最大 `q`
2. **トレランス内**: VMAF >= `target_vmaf - min_vmaf_tolerance` かつサイズ制約内の最大 `q`
3. **エラー**: いずれも該当なしの場合、達成可能な最良 VMAF を報告してエラーを返す

## min_vmaf_tolerance

デフォルト `1.0`。離散的な品質パラメータ空間では目標 VMAF を正確に達成できない場合があるため、このトレランス内であれば許容結果として返す。

## 進捗レポート

`SearchProgress` enum のコールバックにより、以下のイベントを通知する:

| バリアント        | タイミング                                              |
| ----------------- | ------------------------------------------------------- |
| `SampleExtract`   | サンプル抽出中(n / total)                               |
| `Encoding`        | サンプルエンコード中(イテレーション、品質値、n / total) |
| `Scoring`         | VMAF 計測中(イテレーション、品質値、n / total)          |
| `IterationResult` | イテレーション完了(品質値、VMAF、サイズ比)              |
